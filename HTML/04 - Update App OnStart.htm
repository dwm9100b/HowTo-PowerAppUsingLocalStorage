1. Click on App in the left Tree view
  2. Before we get into OnStart, locate the StartScreen property, in the properties dropdown
  3. Enter our one screen: Main Screen
  4. Then select OnStart, yes I know that there are other places to put initializations
  5. Here we will create some of our data structures that will drive the system
  6. This version of the app will be self-contained, no outside dependencies
  7. In order to do this, we will be saving and loading some of our data to/from local storage
  8. First, there is an odd requirement that in order to do a load from local, you need to have a schema defined first
  9. Add the following few lines to the top of your OnStart, easiest to do a copy/paste 
    // Global variable for the Title of our app
Set(gblAppTitle, "My App Title");

// Global variable for the name of our saved data
Set(gblDataFile, "MyTestData");

// Setup the schema for our data collection, required before you can save/load data
ClearCollect(colMenuItems,{intId: 0, strTitle: "", intTime: 0, blnCompleted: false});

// Clear the collection, what remains is the schema, no data
Clear(colMenuItems);

// Now try and load any previously saved data
LoadData(colMenuItems, gblDataFile);

// If the load failed (returned an empty table), then lets populate our data
If (IsEmpty(colMenuItems),

    // Add menu items to our collection, initializing
    ClearCollect(colMenuItems,
        {intId: 1, strTitle: "What is this?", intTime: 0, blnCompleted: false},
        {intId: 5, strTitle: "Prerequisites", intTime: 1, blnCompleted: false},
        {intId: 10, strTitle: "Interesting content", intTime: 4, blnCompleted: false},
        {intId: 15, strTitle: "More good stuff", intTime: 4, blnCompleted: false},
        {intId: 1000, strTitle: "Admin stuff", intTime: 0, blnCompleted: false}
    );
    // End of ClearCollect

    // After we have initialized our data collection, 
    // lets immediately save it to local storage
    SaveData(colMenuItems, gblDataFile);
);
// End of our if IsEmpty
  10. There you have it, we will add one more line at the end later in the tutorial
  11. Some reasoning behind the data structure:
    1. I separate ID's initially by 5 so that later if steps are inserted you don't have to update the other ID's
    2. If intTime is a zero, that item will not be used in calculations nor will there be a Complete button
    3. IntTime should be based on the number of minutes to complete the step
    4. I give the Admin page a large ID so I'll never have to mess with it again
    5. You add as many items to colMenuItems as you have pages/steps in your tutorial/content
