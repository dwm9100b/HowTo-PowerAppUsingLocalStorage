Now that the we have the bulk of the app created and functional, it's time to work on how we'll be presenting the content, which is what this is ultimately all about.

The actual content will be housed in an HTML control, in its Html text property. Now before you get concerned that you don't know HTML, don't worry. I'll be showing you exactly what you'll need to learn and how to apply it. You'll be surprised what you will be able to do.

For this first step, we'll be creating an HTML control as a template with all relevant properties set except for the actual Html text. The HtmlText and BorderThickness will be the only properties that are unique to each HTML control instance. Let's go!

  1. Before we start, make sure you've saved any previous work and have done a publish
  2. Once saved and published, click on conLessonContent in the Tree view
  3. While you are at it, hold the Alt key down, and select the first gallery item, What is this
  4. Then from the top toolbar, click on +Insert
  5. Locate and select the HTML text control, found under the Display section
  6. Let's go ahead and rename it to: htmTemplate
  7. Update its properties as follows

    Property Value
    X Parent.PaddingLeft
    Y 0
    Width Parent.Width - (Parent.PaddingLeft + Parent.PaddingRight)
    Visible galLessonMenu.Selected.intId = Self.BorderThickness
    Size Leave default of 11, you can adjust later if you need
    PaddingTop 0
    HtmlText "This is content for - " & galLessonMenu.Selected.strTitle
    Height Parent.Height
    BorderThickness 1010
    BorderStyle BorderStyle.None

  8. So now we have our template for adding content pages
  9. You will be selecting this template and copying/pasting for each new page
  10. For the purpose of this tutorial, we will be adding 4 copies under conLessonContent
  11. To start, select htmTemplate, over in the … menu, select copy, or Ctrl-C, Mac Cmd-C
  12. Then at the … menu of conLessonContent, click on Paste
  13. Do this 3 more times, should have total of 4 template copies, other than the original
  14. I tend to name the HTML controls close to what its associated menu name is
  15. So we will then rename them as follows
    1. htmWhatIsThis
    2. htmPrerequisites
    3. htmInterestingContent
    4. htmMoreGoodStuff
  16. We don't use borders, so we will be repurposing the BorderThickness property to hold the intId of each page. Update each HTML controls BorderThickness property with its intId

    Control BorderThickness
    htmWhatIsThis 1
    htmPrerequisites 5
    htmInterestingContent 10
    htmMoreGoodStuff 15

  17. Done! You've completed the basic app. The rest would be specific to your particular
    content you want to present. You can now save, publish and run the app.

    Do make note that Saving/Loading data from local storage does not work when you just
    preview vs actually running. So best to go back to the main app list of
    PowerApps and run the app from there.
